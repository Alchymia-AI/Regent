Architecture

Hybrid Mamba-2 with sparse attention

Mamba-2 state space layers interleaved with Grouped Query Attention at an 8:1 ratio. SSM gives constant-memory processing. Attention gives precise token recall.

⚡

Mamba-2 (SSM Layers)

Structured State Space Duality. Input-dependent gating via selective scan. Fixed-size recurrent state regardless of sequence length. Chunked parallel computation for training, sequential scan for inference.

🎯

GQA Attention (Sparse)

Grouped Query Attention with sliding window. Fewer KV heads than query heads for memory efficiency. RoPE position encoding. Placed every ~8 layers as precision recall checkpoints.

🧠

EPG Encoder

Converts structured knowledge graph nodes into dense prefix embeddings. Text features through a small transformer, scalar features (confidence, activation, valence) through learned projections, fused into d_model vectors.

💜

Essence Conditioning

7-dimensional affective state vector projected into hidden layers every N blocks. Modulates model behavior based on emotional state. Same mechanism as timestep injection in diffusion models.

What makes it different

Six things this model does that others do not

Each idea below has shown up somewhere in research. Putting all six into a single production model has not.

01

It scores its own grounding while it writes

The model has two output heads. One produces the next word. The other produces a number between 0 and 1 for that word, where 1 means "this is backed by what I know" and 0 means "I am making this up."

Both heads run on every word in the same pass. Other systems detect hallucinations after the response is finished by re-running the model multiple times or sending the output to a second model. This one knows in real time.

02

It changes how it writes when its confidence drops

The grounding score gates the writing behavior into three modes. Above 0.6 it writes normally. Between 0.3 and 0.6 it slows down and picks safer words. Below 0.3 it stops, looks up relevant facts, and tries again from the point where it got uncertain.

Other models keep writing at the same confidence regardless of whether they are right or wrong.

03

EPG Encoder: it reads structured knowledge as itself, not as text

The Entity Preference Graph holds typed nodes: facts, beliefs, relationships, memories, each tagged with how confident the model is in it, how recently it was used, whether it was a good or bad experience, and what kind of knowledge it is.

Most systems flatten this into English sentences and stuff it into the prompt. This one feeds the nodes directly to the model as compressed inputs the model treats as native knowledge, not a wall of text it has to read and trust.

04

Essence Vector: its mood is a dial, not a paragraph in the prompt

The model takes a 7-number input alongside the conversation. Those numbers describe its current emotional state: how positive it feels, how strongly that feeling should affect output, how much it values truth, civility, kindness, curiosity, and self-preservation.

The numbers get fed in at multiple points during the forward pass, so the whole response is shaped by them. Other models use a persona description that the model has to keep remembering as the response gets longer.

05

It uses constant memory no matter how long the session is

Most language models keep a growing record of every word they have processed. For an 8-hour session that record becomes gigabytes. This model uses different math where the memory of past words is compressed into a fixed-size buffer of about 2 MB.

The buffer does not grow. A drone can run a multi-hour mission without hitting a memory wall. A robot can run a full shift.

06

It runs the grounding check for free

The grounding head is about one-tenth of one percent of the model's total size. It reads the same internal numbers the writing head reads, so it does not need a separate forward pass.

Adding hallucination detection costs almost nothing in compute or memory. Most safety systems double or triple the cost.

Hallucination Mitigation

Built-in grounding verification

The verification head reads the same hidden states as the generation head and outputs a grounding score per token. One forward pass. No separate model, no multiple samples, no post-hoc filtering.

FLOW (score > 0.6)

Model is confident and grounded. Normal sampling with configured temperature, top-p, and top-k. Standard generation for topics backed by knowledge.

CAUTION (0.3 to 0.6)

Model is uncertain. Temperature drops to 0.3, biasing toward conservative tokens. Output shifts to hedged language. Generation slows.

HALT (score < 0.3)

Model is fabricating. Generation stops. System retrieves relevant knowledge, re-injects as context, re-decodes from the halt position.

Per-token, not per-response

Reward models and SelfCheckGPT produce one score for an entire response. The Ver Head scores every token position. This enables catching hallucinations mid-sentence and identifying exactly which spans are fabricated.

Zero extra inference cost

SelfCheckGPT needs 5-20x compute (multiple samples). Constitutional AI needs 2-3 extra LLM calls. The Ver Head is a 2-layer MLP reading existing hidden states. 0.1% of model parameters.

Knowledge-grounded

RAG appends documents to context but models can hallucinate despite having the right documents. The Ver Head is trained specifically on grounded vs fabricated claims relative to the EPG knowledge base.

Applications

Where it fits

Workloads where constant memory, per-token grounding, structured knowledge input, or self-hosting matter more than raw benchmark quality.

Persistent AI Agents

Strong fit

Agents that run for hours or days without restarting. Constant-memory inference means no session limits. EPG input provides persistent context without growing prompts.

Edge / Embodied AI

Strong fit

Robots, drones, wearables with 4-8GB memory budgets. The 7B model at INT4 runs in ~4GB with no memory growth. Indefinite operation on constrained hardware where transformers hit memory walls.

High-Integrity Generation

Strong fit

Medical Q&A, legal document drafting, financial analysis. The Ver Head flags uncertain spans before they reach the user. Per-token scoring, not per-response.

Knowledge-Grounded Systems

Strong fit

Systems with structured knowledge bases beyond document chunks. The EPG encoder ingests typed knowledge nodes with confidence, activation, and category metadata.

Emotionally Adaptive Interfaces

Good fit

Companion AI, therapeutic tools, customer service. Essence conditioning shifts tone and behavior based on a continuous affective state vector, not discrete persona prompts.

Offline / Air-Gapped Deployment

Good fit

Military, industrial, remote environments without internet. Self-contained binary. No API calls, no cloud dependency, no data leaves the device.

Multi-Model Pipelines

Good fit

Use the Ver Head as a hallucination filter in any pipeline. Run a separate LLM for generation, then pass output through Regent's /verify endpoint to get per-token grounding scores and flagged spans.

General-Purpose Chat

Depends on training data

For general chat, instruction following, and creative writing, quality depends on training data and compute. At 7B parameters, it will not match 70B+ models.

Training

Four-phase pipeline

One command runs scrape, tokenize, prepare, and all four training phases. Validated end-to-end on Apple Metal.

# Full automated pipeline: scrape, tokenize, train all 4 phases PYTHONPATH=. python3 scripts/run_pipeline.py \ --config configs/regent_370m.yaml \ --scrape-config pipeline.yaml # Or with synthetic data for validation PYTHONPATH=. python3 scripts/run_pipeline.py \ --config configs/regent_test.yaml \ --synthetic

Phase 1: Base

Standard language modeling. Cross-entropy on next-token prediction. Full model trains on general text corpus. The model learns grammar, world knowledge, reasoning.

Phase 2: Identity

Supervised fine-tuning on domain conversations. Loss on response tokens only. EPG encoder trains jointly. Model learns structured output, knowledge grounding, tone modulation.

Phase 3: Verification

Ver Head training with backbone frozen. BCE loss on grounding labels. Trained on clean, contradicted, entity-swapped, and fabricated pairs. Only 0.1% of parameters update.

Phase 4: Alignment

Direct Preference Optimization. Preferred responses have higher grounding scores and better domain alignment. Frozen reference model. The model learns what "good output" means.

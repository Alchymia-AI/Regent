Architecture

Recurrent backbone with sparse attention

Recurrent layers handle the bulk of processing with a fixed memory footprint. Attention layers are placed every eight blocks to handle precise recall. Two output heads share the same internal state.

Knowledge Nodes ──→ Node Encoder ──→ Prefix Tokens ─┐ │ Behavioral Vector ───────────────────────────────→ │ (injected every 8 layers) │ Token IDs ──→ Embedding ───────────────────────────→ Concatenate │ ▼ ┌───────────────────────────┐ │ Hybrid Backbone │ │ │ │ Layers 1-7: Recurrent │ │ Layer 8: Attention │ │ Layers 9-15: Recurrent │ │ Layer 16: Attention │ │ ... │ │ (57 recurrent + 7 attn) │ └─────────────┬─────────────┘ │ ┌───────┴───────┐ │ │ ▼ ▼ ┌──────────┐ ┌──────────────┐ │ Gen Head │ │ Ver Head │ │ │ │ │ │ → next │ │ → accuracy │ │ token │ │ score [0,1]│ └──────────┘ └──────────────┘
⚡

Recurrent Layers (Mamba-2)

Each layer reads the input and decides how much of it matters, then compresses what it learned into a fixed-size state buffer. That buffer stays the same size whether the session is one minute or eight hours. This is what gives the model constant memory.

🎯

Attention Layers (Sparse)

Placed every eight recurrent layers. Handles precise recall: finding a specific fact from earlier in the session that the recurrent layers may have compressed away. Operates over a sliding window to keep memory bounded.

🧠

Knowledge Graph Encoder

Takes structured knowledge nodes (facts, beliefs, relationships, memories) and converts them into inputs the model reads as native context. Each node carries metadata: confidence level, recency, emotional weight, category. No text serialization required.

💜

Behavioral Conditioning

A 7-number vector (mood, influence, truth bias, civility, good/evil, curiosity, self-preservation) is fed into the model at multiple points during generation. The whole response is shaped by it. Not a prompt prefix that the model has to keep re-reading.

What makes it different

Six things this model does that others do not

Each idea below has shown up somewhere in research. Putting all six into a single deployable model has not.

01

It scores its own accuracy while it writes

The model has two output channels. One produces the next word. The other produces a number between 0 and 1 for that word, where 1 means "I am confident this is correct" and 0 means "I am guessing."

Both channels run on every word in the same pass. Other systems detect errors after the response is finished by re-running the model or sending the output to a second one. This one knows in real time.

02

It changes behavior when its confidence drops

The accuracy score gates the generation into three modes. Above 0.6 it writes normally. Between 0.3 and 0.6 it slows and picks more conservative words. Below 0.3 it stops, retrieves relevant facts from its knowledge store, and tries again from the point where it got uncertain.

Other models write at the same pace and confidence whether they are right or wrong.

03

It reads structured knowledge as itself, not as text

The knowledge graph encoder takes typed nodes directly as input. Facts, relationships, memories, constraints, each with confidence scores and category labels. The model treats them as native knowledge.

Other systems flatten knowledge into English sentences and inject them as prompt text. This one does not pay the token cost to re-inject the same knowledge on every call.

04

Its behavioral state is a dial, not a paragraph

The model takes seven numbers alongside the conversation. They describe its current state: how positive it feels, how strongly that should affect output, how much it values truth, civility, kindness, curiosity, and self-preservation.

Those numbers are injected into the model at multiple layers. The entire response is shaped by them continuously. Other models use a persona description in the prompt that the model has to keep remembering.

05

It uses constant memory no matter how long the session is

Standard AI models keep a growing record of every word they have processed. For a multi-hour session that record becomes gigabytes. Regent compresses past context into a fixed-size buffer of about 2 MB for the 7B model.

The buffer does not grow. A drone runs a six-hour mission on the same memory it used in the first second. A robot works a full shift. A long patient encounter does not degrade performance.

06

Accuracy scoring adds almost nothing to the cost

The accuracy-checking component is one-tenth of one percent of the model's total size. It reads the same internal state the writing head reads, so no extra computation is needed.

Most safety systems require running the model two to twenty times to get a reliability signal. This one is already running.

Accuracy Scoring

Built-in grounding verification

The verification head reads the same internal state as the generation head and outputs an accuracy score per word. One pass. No separate model, no multiple samples, no post-hoc filtering.

FLOW (score > 0.6)

Model is confident and grounded. Writes normally at the configured temperature and sampling settings. Standard generation for topics the model knows well.

CAUTION (0.3 to 0.6)

Model is uncertain. Temperature drops, output shifts toward conservative and hedged language. The response signals its own uncertainty rather than asserting it.

HALT (score < 0.3)

Model is fabricating. Generation stops. The system retrieves relevant knowledge, re-injects it as context, and re-generates from the point where confidence broke down.

Per-word, not per-response

Most accuracy checks produce one score for an entire response. This model scores every word. That means catching a fabricated claim mid-sentence and knowing exactly which span is the problem, not just that the response has one somewhere.

No extra inference cost

Sampling-based verification requires running the model 5 to 20 times. Constitutional AI methods require 2 to 3 extra calls. The verification head is a small network reading state that already exists. 0.1% of model parameters, zero extra passes.

Grounded in the knowledge base

Standard retrieval augmentation gives the model the right documents but does not prevent it from ignoring them. The verification head is trained specifically to distinguish between claims grounded in the knowledge base and claims that are not.

Deployment

Three ways to deploy

Export from the Model Studio interface or the command line. One click to HuggingFace. One click to Docker. Runs on NVIDIA CUDA and Apple Silicon.

🤗

HuggingFace

Export to a HuggingFace-compatible package and load it with two lines of code using any tool in the HuggingFace ecosystem. Push directly to HuggingFace Hub from the interface. Works with existing fine-tuning pipelines, quantization tools, and deployment frameworks.

from transformers import AutoModelForCausalLM, AutoTokenizer model = AutoModelForCausalLM.from_pretrained( "path/to/export", trust_remote_code=True )
🐳

Docker / Self-Hosted

Export as a self-contained Docker package with a FastAPI inference server included. Runs on any machine with Docker installed. No Python environment setup required. API-compatible with standard chat completion endpoints.

⚡

CUDA + Apple Silicon

Runs natively on NVIDIA GPUs and Apple Silicon. The CUDA path includes an optional Triton kernel for maximum throughput on long sequences. The Apple Silicon path uses Metal for native acceleration. CPU fallback works on any machine.

🖥

Model Studio Interface

A browser-based interface ships with the model. From it you can scrape and prepare training data, run all four training phases with live monitoring, review checkpoints, chat with the running model and see per-word accuracy scores in real time, manage the knowledge graph, and export when ready. No command line required.

📦

Weight Formats

Weights are stored in SafeTensors format, readable by both PyTorch and MLX without conversion. Export supports float32, float16, and bfloat16. The same weight file loads in HuggingFace Transformers on CUDA and in MLX on Apple Silicon.

Quick Start

Up and running in three steps

Clone, install, and start. The Model Studio interface opens in your browser and handles everything from there.

Step 1 — Clone and install

# Apple Silicon git clone https://github.com/Alchymia-AI/Regent cd regent-model python3.12 -m venv .venv && source .venv/bin/activate pip install mlx pyyaml sentencepiece numpy fastapi uvicorn pydantic # NVIDIA GPU pip install torch --index-url https://download.pytorch.org/whl/cu124 pip install pyyaml sentencepiece numpy fastapi uvicorn pydantic safetensors transformers

Step 2 — Start the interface

# Starts Model Studio UI + API server ./start.sh # Opens at http://localhost:3000 # API at http://localhost:8400 # With a specific model config and weights ./start.sh --config configs/regent_7b.yaml \ --model checkpoints/alignment/regent.safetensors

Step 3 — Or load from HuggingFace

from transformers import AutoModelForCausalLM, AutoTokenizer import torch model = AutoModelForCausalLM.from_pretrained( "alchymia-ai/regent-7b", trust_remote_code=True, torch_dtype=torch.float16, device_map="auto", ) tok = AutoTokenizer.from_pretrained( "alchymia-ai/regent-7b", trust_remote_code=True ) out = model.generate( **tok("Hello, Regent!", return_tensors="pt").to(model.device), max_new_tokens=200 )
No weights yet? Run with --synthetic to validate the architecture on random data. All four training phases run end-to-end. When you have a real corpus, swap it in and run again.
Applications

Where it fits

Workloads where constant memory, per-word accuracy scoring, structured knowledge input, or self-hosting matter more than raw benchmark rankings.

Legal Research and Drafting

Strong fit

Every claim in a brief needs to be traceable. The accuracy score gives a per-sentence confidence signal before the document leaves the desk. Case law and statute are graph-shaped; the knowledge encoder ingests them directly rather than as pasted text.

Healthcare and Clinical Support

Strong fit

Long patient sessions require memory that does not degrade with length. Hallucination has clinical consequences. UMLS, SNOMED, and ICD are structured ontologies the knowledge encoder consumes natively. The accuracy score provides the grounding signal clinical workflows require.

Robotics and Drones

Strong fit

A robot running an 8-hour shift and a drone on a 6-hour mission both need a cognitive layer that does not accumulate memory. The 7B model at INT4 runs in 4GB on a Jetson Orin. The accuracy score gates actions before they are executed.

Government and Defense

Strong fit

Air-gap deployment is a hard requirement that eliminates API-based competitors. The model ships as weights and runs on local hardware with no external connectivity. The accuracy score produces auditable records. Sovereign AI procurement is a stated priority across allied governments.

Financial Research and Risk

Strong fit

Long analyst sessions over structured market data. Risk teams need every claim traceable before it influences a decision. The accuracy score is the audit trail. Self-hosting removes data sovereignty concerns.

Persistent AI Agents

Strong fit

Agents that run for hours or days without restarting. Constant memory means no session limit. Knowledge graph input provides persistent context without growing prompts. Behavioral conditioning stays consistent across the full session.

General-Purpose Chat

Depends on training data

For general chat and creative writing, quality depends on training data and compute. At 7B parameters, it will not match 70B+ frontier models on open-ended tasks. This is not the target use case.

Emerging Market Deployment

Strong fit

Organizations in markets where per-token cloud pricing is not economically viable. One license, local hardware, zero marginal cost per inference. Works without internet. The 7B model runs on a $500 edge device. Healthcare, legal aid, agricultural, and financial inclusion organizations across Africa, South Asia, Southeast Asia, and Latin America.

Training

Four-phase pipeline

Run all four phases from the Model Studio interface with one click, or from the command line with one command. Validated end-to-end on Apple Silicon and NVIDIA CUDA.

# Command line: full pipeline from scrape to alignment PYTHONPATH=. python3 scripts/run_pipeline.py \ --config configs/regent_370m.yaml \ --scrape-config pipeline.yaml # Or open the Model Studio UI and click Start ./start.sh # opens at http://localhost:3000

Phase 1: Base

Language modeling on a general text corpus. The model learns grammar, world knowledge, and reasoning. Full model trains. This is the foundation all other phases build on.

Phase 2: Identity

Supervised fine-tuning on domain conversations and structured knowledge examples. The knowledge encoder trains jointly. The model learns the output format and tone expected for the deployment context.

Phase 3: Verification

Accuracy head training with the backbone frozen. Trained on examples of grounded output alongside fabricated, contradicted, and entity-swapped versions. Only the small accuracy head updates. 0.1% of parameters.

Phase 4: Alignment

Preference learning against a frozen reference copy. Responses with higher accuracy scores and better domain alignment are preferred. The model learns what constitutes good output for its specific use case.
